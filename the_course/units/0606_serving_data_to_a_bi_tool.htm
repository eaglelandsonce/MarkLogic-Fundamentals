Serving Data to a BI Tool

Steps can be done to serve data from a MarkLogic database as rows, so that business intelligence tools can make SQL queries against them. This is accomplished by defining a template, defining an ODBC server port, and installing the MarkLogic ODBC driver.

Template

Within a template, criteria for row data is specified. For any document content that matches the template context and column paths, the view is populated with rows.


By creating and inserting a template, a view is generated, containing rows that SQL queries can be run against.


ODBC Server

For being able to make SQL queries from outside of MarkLogic, an ODBC server port needs to have been created. This can be done via the Admin Interface or programmatically.

MarkLogic ODBC Driver

On the server where the business intelligence tool (e.g. Tableau, Qlik, Power BI, etc.) is installed, the MarkLogic ODBC driver can be installed and configured, allowing SQL queries to go to the MarkLogic database from the business intelligence tool, via the ODBC server port.


Detail on the steps for making data available to business intelligence tools is in the SQL Data Modeling Guide at docs.marklogic.com/guide/sql, as well as hands-on in the MarkLogic Data Integration course, available at www.marklogic.com/learn/courses/marklogic-data-integration.
