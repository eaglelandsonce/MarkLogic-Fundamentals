Reading via the REST API

A document can be read via the MarkLogic REST API by using the /v1/documents endpoint and specifying the URI of the document to read.

curl --anyauth --user admin:admin -X GET "http://localhost:8060/v1/documents?uri=/patents/waterski_drone.json"

By having database documents available via the REST API, pretty much any programming language (i.e. any that can make HTTP requests) can be used to read data from a MarkLogic database.

Writing via the REST API

A document can be written via the MarkLogic REST API by using the /v1/documents endpoint and specifying a URI and content for the document.

curl --anyauth --user admin:admin -X PUT -i -H "Content-Type: application/json" -d '{"patent": {"title": "﻿Smart tennis racket", "inventor": "﻿Ashutosh", "description": "﻿AI-powered tennis racket that dynamically calculates the incoming ball speed to automatically adjust the elasticity of the strings for an optimum shot"}}' "http://localhost:8060/v1/documents?uri=/patents/smart_racket.json"

Creating a Custom Endpoint

Along with being able to use the built-in REST API endpoints, custom endpoints can also be created (e.g. microservices).

curl --anyauth --user admin:admin -X GET -i -H "Content-type: application/json" http://localhost:8025/v1/resources/inventions?rs:inventor=Ashutosh

Detail on creating custom endpoints is in the Extending the REST API chapter of the REST Application Developer's Guide at docs.marklogic.com/guide/rest-dev/extensions.
