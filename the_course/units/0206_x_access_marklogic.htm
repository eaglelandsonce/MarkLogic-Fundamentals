Exercise: Access MarkLogic

In this walkthrough exercise, you access MarkLogic using a few of the interfaces.

 

  1. To use Query Console, open a new browser window in the virtual machine, and enter the following URL:

    http://localhost:8000
  2. If prompted, enter admin and admin, to log in to the application.
  3. In the Database drop-down list, view the list of databases that already exist, created when you installed MarkLogic:


  4. With the Documents database selected, click the Explore button.
  5. In the lower panel, see that there are currently 0 documents in the Documents database:


  6. In the Query Type drop-down list, notice the query languages that can be used in Query Console:

  7. If desired, enter the following and, with JavaScript selected, press Run:

    cts.search('potato');
  8. Notice the response:

  9. Realize this is because there were not any documents in the database with the word potato in them.
  10. If desired, in a new browser window within the virtual machine, enter localhost:8002/history.
  11. If prompted, enter admin and admin, to log in to the application.
  12. See that you are viewing the Monitoring History dashboard, viewing resource activity for the past day:


    Note: If you see a message of 'No resources to display' or 'No data to display', know that you can view the dashboard later, when there is data to display
  13. If desired, click the 1h shortcut link, to view activity for the past hour:


  14. Close the Monitoring History dashboard.
  15. Open a command prompt window (Applications - System Tools - Terminal).
  16. Enter the following (can copy from mls-fundamentals/unit02/ex06a.txt, and right-click to select Paste), to use the REST API to see the Documents database status:

    curl --anyauth --user admin:admin -X GET http://localhost:8002/manage/v2/databases/Documents?view=status
  17. Once the command completes, scroll back up, to the beginning of the response, to see that you are viewing status information for the Documents database:

  18. Realize that you can use the REST API to do all sorts of activities including creating databases, writing documents, reading documents, creating users, etc.
  19. If desired, visit docs.marklogic.com/REST to learn about other REST API endpoints.
