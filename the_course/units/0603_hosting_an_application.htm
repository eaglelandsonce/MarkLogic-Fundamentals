Hosting an Application

If using XQuery as the programming language for an application, the application can be hosted by MarkLogic by creating an HTTP server.

HTTP Server

A MarkLogic HTTP server includes the capabilities of both an application server and a web server.

An HTTP server can be created programmatically or via the Admin Interface. To create an HTTP server via the Admin Interface, the Configure - Groups - Default - App Servers - Create HTTP tab can be used.


The database field is for specifying which database the application server is for.

The modules field is for specifying whether the application files will live on the file system or in a database.

The root field is for specifying the location of the application files.

The port field is for specifying which port the application will be available on.

The server name field is for giving the HTTP server a name.

Viewing a Hosted Application

Once the fields have been filled out and the ok button clicked, the application server is available, with the port using the application files specified in the root field.

Understanding App Server Types

MarkLogic App Servers provide an interface for communicating with MarkLogic databases, as well as, in the case of a MarkLogic HTTP server, the option of referencing application files for a web application.

The type of servers that can be defined via the tabs are: HTTP (for a web application and/or REST API interface), WebDAV (for dragging and dropping documents into a database), XDBC (for low-level Java and .NET connections through XCC), and ODBC (for business intelligence tools like Tableau or Power BI to make SQL queries to a MarkLogic database).
