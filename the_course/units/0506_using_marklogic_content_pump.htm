Using MarkLogic Content Pump

MarkLogic Content Pump (MLCP) is a popular Java-based command line tool that is highly scalable, commonly used to bulk load files into a MarkLogic database.

Setting up MLCP

As a Java-based command line tool, MLCP requires JRE 1.8 or later.

MLCP can be downloaded for free from developer.marklogic.com/products/mlcp. Once it has been downloaded, it can be unzipped to anywhere on the file system.

For MLCP to be able to communicate with a MarkLogic database, there needs to be a REST API instance port or an XDBC server port for the database.

Using MLCP

To use MLCP, a command can be run on the command line, from the directory where MLCP is located.

For example, the following could be run in a command prompt window, to bulk upload documents from the file system to the patents database via port 8060.

mlcp.sh import -host localhost -port 8060 -username admin -password admin -mode local -input_file_path Desktop/mls-fundamentals/data/patents -output_uri_replace "home/cent/Desktop/mls-fundamentals/data/patents, 'patents'" -output_permissions patents-user-role,read,patents-admin-role,update

If running MarkLogic on Windows, mlcp.bat would be used instead of mlcp.sh:

C:\mlcp\bin\mlcp.bat import -host localhost -port 8060 -username admin -password admin -mode local -input_file_path C:\mls-fundamentals\data\patents -output_uri_replace "C:/mls-fundamentals/data/patents, 'patents'" -output_permissions patents-user-role,read,patents-admin-role,update

Loading Triples

To load triples via MLCP, the -input_file_type RDF option can be used.

~/Desktop/mlcp/bin/mlcp.sh import -mode local -host localhost -port 8060 -username admin -password admin -input_file_path ~/Desktop/mls-fundamentals/data/triples/patents.nt -input_file_type RDF -output_uri_prefix /triplestore/

To learn about triples, see the MarkLogic Semantics course, available at www.marklogic.com/learn/courses/using-marklogic-semantics.
