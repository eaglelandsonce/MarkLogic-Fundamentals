Accessing MarkLogic

There are many interfaces for communicating with MarkLogic.

Common interfaces for monitoring MarkLogic include:

  • Admin Interface
  • Monitoring Dashboard
  • Monitoring History Dashboard
  • REST API
  • Log files


Common interfaces for making configuration changes include:

  • Admin Interface
  • REST API
  • ml-gradle
  • Query Console
  • Admin API


Common interfaces for ingesting data include:

  • MarkLogic Data Hub
  • Apache NiFi
  • MarkLogic Content Pump
  • Data Movement SDK
  • Java API
  • Node.js API
  • JavaScript
  • XQuery
  • Query Console
  • REST API
  • ml-gradle
  • XCC


Common interfaces for querying data include:

  • REST API
  • Java API
  • Node.js API
  • JavaScript
  • XQuery
  • Query Console
  • SQL
  • MarkLogic ODBC Driver
  • SPARQL
  • XCC


The Admin Interface provides a graphical user interface for a multitude of administrative tasks, such as creating and configuring databases, forests, application servers, security, backups, schemas, and groups.

The Admin API provides a way to script tasks otherwise available in the Admin Interface.

Apache NiFi can be used to ingest data from a relational database into a MarkLogic database.

The Data Movement SDK, which is a part of the Java API, is a set of Java classes for ingesting and transforming documents.

Built on the REST API, the Java API provides Java classes to make it easy for Java developers to build applications that use a MarkLogic database.

JavaScript is a native programming language in MarkLogic, useful for a variety of tasks.

MarkLogic Content Pump (MLCP) is a highly-efficient command line tool for loading large numbers of documents.

MarkLogic Data Hub is a platform that runs on MarkLogic Server, designed for loading and curating data which has come from a wide variety of sources.

The MarkLogic ODBC Driver provides a way for any applications that support ODBC connectivity to be able to make SQL queries to a MarkLogic database.

ml-gradle is a Gradle plug-in that can be used to automate MarkLogic tasks (e.g. deploying a MarkLogic project).

The Monitoring Dashboard and Monitoring History Dashboard are GUI tools for monitoring MarkLogic.

Built on the REST API, the Node.js API provides an npm package for Node.js developers building applications that use a MarkLogic database.

Query Console is an interface that allows developers to communicate with MarkLogic by running code written in JavaScript, XQuery, SPARQL, or SQL.

The REST API provides a programming language-agnostic interface for communicating with MarkLogic.

SQL queries can be run against data that has been exposed as rows.

XCC (XML Content Connector) is an older, low-level API, previously used by Java and .NET developers. (Nowadays, Java developers use the Java API, and .NET developers use the REST API.)

XQuery is a native programming language in MarkLogic, and can be used to create applications hosted by MarkLogic.
