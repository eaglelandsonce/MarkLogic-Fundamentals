Understanding Indexes

Indexes are MarkLogic's secret sauce for being able to execute queries in real-time even when there are millions (or billions or trillions) of documents in a database.

At a minimum, MarkLogic provides baseline indexing of the words and structure of every document that is loaded into a MarkLogic database. As a document is being loaded, MarkLogic creates an index of all of the words in the document, as well as the structure of the document. This not only allows for real-time search for words in the documents, but also real-time search based on paths in the documents. (For example, searching only in the song title or artist element of a song document.)

Along with the baseline indexing of a document's words and structure, additional indexes can be defined for a database, to speed up the performance of an application. Which indexes would help the performance of a given application depend on the application itself. For example, in the Star Wars application, having an index for the Name values and an index for the Role values would enable sorting to happen in real-time even if there were millions of documents in the database.

In addition, some MarkLogic search features, such facets and semantics, require indexes.

Optional Indexes

For each database, there are a number of optional indexes available.

To turn on an index, the true button can be selected and then the ok button pressed.

Adding an index does not require a restart or system delay. As an index is being populated, MarkLogic re-indexes the database in the background. While it is re-indexing in the background, any queries to the database continue to be served. Once the index is available, any new queries take advantage of the index if it is useful to the query.

As adding indexes takes up additional disk space, and can make the loading of documents take longer, the recommendation is to proceed with the default indexes first, and then add additional indexes as needed.

Detail on the indexes is available in the documentation or by selecting the Help tab.

Defining Range Indexes

A range index can be defined that contains the values of a JSON property or an XML element from documents in a database. For example, a range index could be set up for all of the name values in the documents in the star-wars database.

By creating a range index of all the values in the name property of the characters documents, the Name sort can happen in real-time, even if there are millions of documents in the database.

Indexes can be configured via the Admin Interface or programmatically via the APIs.
