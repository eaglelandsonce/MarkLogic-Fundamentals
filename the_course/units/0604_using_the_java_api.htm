Using the Java API

Built on the REST API, the Java API provides a native interface for Java developers to create applications that communicate with a MarkLogic database.

REST API Instance

In order to be able to use the Java API to communicate with a MarkLogic database, a REST API port needs to exist for the database.

Getting the Java API Files

The Java API files can be acquired from the MarkLogic Community site (developer.marklogic.com/products/java), or from GitHub (github.com/marklogic/java-client-api), or via build tools Maven or Gradle.

Learning Resources

The Java Application Developer's Guide (docs.marklogic.com/guide/java.pdf) is a great resource for Java developers to use the Java API for communicating with a MarkLogic database.

Also, Java API tutorials exist on the MarkLogic Community site, at developer.marklogic.com/learn/java and developer.marklogic.com/try/java/index.

In addition, there is a 3-day MarkLogic Java Developer course, available at www.marklogic.com/learn/courses/developing-marklogic-apps-java.

Moreover, the Samplestack reference instance (github.com/marklogic/marklogic-samplestack) provides a 3-tier, enterprise reference architecture for Java developers to reference when creating their own applications. Implemented using Angular, Gradle, Spring, Java, and MarkLogic, Samplestack includes examples of enterprise-grade features such as multi-statement transactions, role-based security, and LDAP integration.
