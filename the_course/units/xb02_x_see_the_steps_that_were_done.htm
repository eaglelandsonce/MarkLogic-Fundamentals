Exercise: See the Steps That Were Done

In this exercise, you view the steps that were done to create the CentOS 7 virtual machine for this course. Realize that, if you go to reproduce these steps, some of the versions and URLs may have moved on.

Note: It is not necessary to build the virtual machine yourself. You can download it from MarkLogic University.

 

  1. Downloaded and installed VirtualBox (VirtualBox-5.2.34-133893-OSX.dmg).
  2. Downloaded CentOS ISO (CentOS-7-x86_64-Minimal-1908.iso).

To create the virtual machine:

  1. In VirtualBox, clicked the New button.
  2. For Name, entered centos7_64.
  3. For Type, left Linux selected.
  4. For Version, left Red Hat (64 bit) selected.
  5. Clicked the Continue button.
  6. For Memory size, entered 4096 (4 GB), and clicked the Continue button.
  7. Left Create a virtual hard drive now selected, and clicked the Create button.
  8. Left VDI (VirtualBox Disk Image) selected, and clicked the Continue button.
  9. Left Dynamically allocated selected, and clicked the Continue button.
  10. For File location and size, left the default name of centos7_64, left the default location (/Users/rstryker/VirtualBox VMs/centos7_64), and entered 20 GB.
  11. Clicked the Create button.

To increase the performance of the virtual machine, specified 2 CPUs:

  1. In VirtualBox, with the virtual machine selected, clicked the Settings icon.
  2. Clicked the System category.
  3. Clicked the Processor tab.
  4. For Processor(s), entered 2.
  5. Clicked OK.

To use a CentOS 7 ISO image for the virtual machine:

  1. With the virtual machine selected on the left, clicked the Settings icon.
  2. Clicked the Storage category.

  5. Navigated to select the CentOS ISO downloaded in Step 2.
  6. Clicked the Open button and clicked the OK button.

To turn off transparent huge pages, set the I/O scheduler to deadline, and set huge pages to 3/8 system memory (virtual machine has 4 GB RAM, so 4096 * 0.375 = 1536, and then 1536 / 2.048 = 750 huge pages):

  1. Selected Applications - System Tools - Terminal to open a terminal window.
  2. Entered su to be root.
  3. Entered centos when prompted for password.
  4. Entered vim /etc/default/grub.
  5. Pressed the i key to be able to edit the file.
  6. At the end of GRUB_CMDLINE_LINUX, after quiet, entered:

     transparent_hugepage=never elevator=deadline
  7. Pressed the Esc key.
  8. Typed :wq and then pressed the Enter key to exit the file.
  9. To regenerate the boot configuration, entered:

    grub2-mkconfig –o /boot/grub2/grub.cfg
  10. To be able to set the number of huge pages to 3/8 of the system memory, entered vim /etc/sysctl.conf.
  11. Pressed the i key to be able to edit the file.
  12. Scrolled to the end of the file.
  13. Added vm.nr_hugepages=750.
  14. Ensured that there was not a comment character (#) at the beginning of the line.
  15. Pressed the Esc key.
  16. Typed :wq and then pressed the Enter key to exit the file.
