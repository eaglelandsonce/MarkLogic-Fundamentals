Understanding MarkLogic

MarkLogic, formally known as MarkLogic Server, is an enterprise NoSQL database that is commonly used for creating a data hub.

NoSQL database features (in MarkLogic, in most NoSQL database technologies):

  • No schema required
  • Horizontal scalability
  • High availability

MarkLogic includes broad support for data, unstructured to structured, including JSON, XML, RDF (for semantic triples), text, and binary data types.

Enterprise features (in MarkLogic, not in most other NoSQL database technologies):

  • ACID transactions
  • Certified security
  • Built-in search

Although MarkLogic differs from a relational database in that MarkLogic was designed to not have schema requirements for data, and was designed for huge horizontal scalability on commodity hardware, it is similar to relational databases in that it is for mission-critical, real-time applications.

MarkLogic Data Hub

MarkLogic Data Hub is a platform that can be installed on MarkLogic Server, providing a data integration framework for importing and curating data from a wide variety of sources (e.g. silos).

Along with MarkLogic Data Hub's data loading and curation capabilities are the ability to preserve data lineage, do entity mapping, do smart mastering, and control data security.

Documentation on MarkLogic Data Hub is available at docs.marklogic.com/datahub.
