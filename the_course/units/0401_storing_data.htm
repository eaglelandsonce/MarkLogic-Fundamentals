Storing Data

Data is stored in a MarkLogic database as documents, triples, or both.

Document

Modeling data as a document is the most common way to store data in MarkLogic. In MarkLogic, a schema is not required.

Example JSON document:

{
 "orderNum": "10072",
 "orderDate": "2017-01-14",
 "items":
   [{
      "item":
      {
        "product": "SpeedPro Ultimate",
        "price": 999,
        "quantity": 1
      }
   },
   {
      "item":
      {
        "product": "Mens Racer Helmet",
        "price": 95,
        "quantity": 1
      }
   }],
 "shipTo":
   {
      "firstName": "Dave",
      "lastName": "Delansantos",
      "company": "Rolling Fast Deliveries",
      "street": "800 Market Street",

      "zipcode": "94111",
      "phone": "415-787-1212",

   },

   {
      "firstName": "Graham",
      "lastName": "Warman",

Envelope Technique

For storing additional data about a document, it is common to 'envelope' a document, so that the source data can be preserved, and yet also allow for additional data to be stored in the document (e.g. metadata).

{  
 "envelope":
   {
     "extraContent": {...},
     "source":
       {
         "orderNum": "10072",
          ...
       }
   }
}

Triple

Triples are useful for describing relationships between entities.



SPARQL queries can be run across triples (e.g. to make interesting discoveries).
