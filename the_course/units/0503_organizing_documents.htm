Organizing Documents

A subset of documents in a database can be addressed through the use of directories (which are based on the URIs of documents) or collections.

Directories

Directories come from document URIs. For example, if loading recipe documents, each document could be assigned a URI that starts with /recipes/.

/recipes/tofu-soup.json

Example JavaScript for referencing /recipes/ documents:

xdmp.directory('/recipes/')

Collections

Collections provide a way to tag a document.

Example collection assignment:

xdmp.documentAddCollections('/recipes/tofu-soup.json', ['vegetarian', 'spicy'])

Example JavaScript to reference the documents in a collection named vegetarian:

fn.collection('vegetarian')

A document can belong to more than one collection, though it is recommended to not have a document belong to more than 100 collections. (MarkLogic can handle millions of collections in a database without difficulty.)

Detail on the use of collections is at docs.marklogic.com/guide/search-dev/collections.
