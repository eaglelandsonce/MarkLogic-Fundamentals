Bringing In Documents

There are many ways to insert documents into a MarkLogic database. Interfaces available include:

  • MarkLogic Data Hub
  • Apache NiFi
  • MarkLogic Content Pump
  • Data Movement SDK (Java API)
  • Apache Kafka
  • MuleSoft
  • Node.js API
  • REST API
  • JavaScript functions
  • XQuery functions
  • XCC
  • WebDAV
  • Content Processing Framework



MarkLogic Data Hub can be used to import, as well as harmonize, data.

Apache NiFi is often used when needing to ingest data from a relational database into a MarkLogic database.

MarkLogic Content Pump is a command-line tool for bulk loading billions of documents into a MarkLogic database.

Included in the Java API, the Data Movement SDK provides classes for Java developers to use to import and transform documents.

Apache Kafka can be used to stream data into a MarkLogic database via the Kafka-MarkLogic-Connector.

Built on the Data Movement SDK, the MarkLogic Connector for MuleSoft can be used to bring data from a variety of other systems into a MarkLogic database.

The Node.js API provides Node.js classes for developers to use to write documents to a MarkLogic database from their Node.js code.

The REST API provides a programming language-agnostic way to write documents to a MarkLogic database.

JavaScript functions can be used to write documents to a MarkLogic database from Query Console or from a JavaScript application.

XQuery functions can be used to write documents to a MarkLogic database from Query Console or from an XQuery application.

XCC is a low-level interface that allows Java or .NET applications to write documents to a MarkLogic database.

WebDAV can be used to drag and drop documents into a MarkLogic database.

The Content Processing Framework is a pipeline framework for making changes to documents as they are being loaded into a database, such as enriching the data, or transforming a PDF or MS Office document to XML.
