Accessing Data

There are many ways to access data in a MarkLogic database. Programming interfaces available for accessing data include:

  • REST API
  • Java API
  • Node.js API
  • JavaScript
  • XQuery
  • XCC
  • SPARQL
  • SQL
  • Data Services


The REST API provides a programming language-agnostic way to read documents from a MarkLogic database.

The Java API provides an interface for Java developers to communicate with a MarkLogic database.

The Node.js API provides an interface for Node.js developers to communicate with a MarkLogic database.

As a native language in MarkLogic, JavaScript can be used to read documents from a MarkLogic database, as well as serve data via custom REST API endpoints.

As a native language in MarkLogic, XQuery can be used to read documents from a MarkLogic database, as well as serve data via custom REST API endpoints, as well as create applications that can be hosted by MarkLogic.

XCC is an older, low-level interface that Java and .NET applications used in the past to communicate with a MarkLogic database. (Nowadays, the Java API and the REST API are commonly used in place of XCC.)

SPARQL is common for accessing triples data.

By exposing document data as rows, SQL queries can be made.

Data Services (a.k.a. microservices) can be created to make it easy for developers outside of MarkLogic to make custom data calls. For example, providing a Java developer with custom methods for accessing data, without the Java developer needing to understand MarkLogic.
