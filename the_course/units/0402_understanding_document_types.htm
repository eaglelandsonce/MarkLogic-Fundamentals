Understanding Document Types

Document types are JSON, XML, text, and binary.

JSON and XML documents are especially interesting, as queries can be done not only for the existence of text, but also based on document structure.

JSON document example:

{
 "recipe":
  {
    "name": "tofu soup",
    "ingredients": ["tofu block", "carrots", "mushrooms", "green onions", "coconut milk"],
    "directions":
     {
       "preparation": "Chop ingredients into medium-size cubes.",
       "cooking":
       {
         "step1": "In a large pot, combine all ingredients.",
         "step2": "Cook on medium heat for 45 minutes."
       }
     }
  }
}

Example (JavaScript) search:

cts.search(cts.jsonPropertyWordQuery('ingredients', 'carrots'));


XML document example:

<recipe>
  <name>tofu soup</name>
  <ingredients>
    <ingredient>tofu block</ingredient>
    <ingredient>carrots</ingredient>
    <ingredient>mushrooms</ingredient>
    <ingredient>green onions</ingredient>
    <ingredient>coconut milk</ingredient>
  </ingredients>
  <directions>
    <preparation>Chop ingredients into medium-size cubes.</preparation>
    <cooking>
      <step1>In a large pot, combine all ingredients.</step1>
      <step2>Cook on medium heat for 45 minutes.</step2>
    </cooking>
  </directions>
</recipe>

Example (XQuery) search:

cts:search(fn:doc(), cts:element-word-query(xs:QName("ingredient"), "carrots"));

Note: Although it is common to use JavaScript with JSON files, and XQuery with XML files, it is also possible to mix and match (e.g. JavaScript with XML, XQuery with JSON).


Text documents can be searched for the existence of text, but not based on document structure.

The ingredients for tofu soup are tofu block, carrots, mushrooms, green onions, and coconut milk. Chop all of the ingredients into medium-size cubes. Then, to cook, in a large pot, combine all ingredients, and cook on medium heat for 45 minutes.

Example search:

cts.search(cts.wordQuery('carrots'));


With MarkLogic's support for binary files, pretty much any document type (e.g. images, sounds, video, etc.) can be stored in a MarkLogic database.

The contents of binary files cannot be searched, but any metadata stored in the properties of any type of document can be searched.

Using MarkLogic's conversion functions, documents in some binary formats (e.g. PDF, Word, Excel, PowerPoint, and HTML) can be converted to XML, which allows for searching the document's content, including by structure.
