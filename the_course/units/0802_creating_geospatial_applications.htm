Creating Geospatial Applications

MarkLogic offers geospatial capabilities, used in a wide range of applications, with the ability to query points, polygons, and other regions.

Examples

For example, MarkLogic's geospatial capabilities can be used for querying mortgages.

Sensing Clues uses MarkLogic's geospatial capabilities to help prevent the illegal poaching of animals. For detail, see the Geospatial Intelligence Saving Wildlife video recording at www.marklogic.com/solutions/industry/public-sector/geospatial.

Yell uses MarkLogic's geospatial capabilities for users to be able to locate nearby services.

Latitude and Longitude

Latitude and longitude values can be defined within JSON and XML documents, providing a way to associate the document's content with a location.

Geospatial standards supported include the common WGS84 coordinate system, ETRS89, and raw. Detail on coordinate systems is at docs.marklogic.com/guide/search-dev/geospatial#id_98149.

Querying

Geospatial queries can be made via the extensive cts geospatial functions.

For example, a query could be written to find any documents that have latitude and longitude values within a 30-mile radius of a given point (e.g. downtown Vancouver).

﻿cts.search(
cts.jsonPropertyPairGeospatialQuery("location", "lat", "long",
cts.circle(30, cts.point(49.2745813, -123.1240661))));

Index

It is common to define a geospatial index, for high performance when making geospatial queries.

The index to use is based on the structure of the latitude and longitude values in the documents.
