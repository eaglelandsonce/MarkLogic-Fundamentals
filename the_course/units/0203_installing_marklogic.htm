Installing MarkLogic

MarkLogic can be installed on CentOS via a command prompt window, as root, using yum.

yum install MarkLogic-10.0-3.x86_64.rpm

Configuring Which User MarkLogic Will Run As in CentOS

By default, on UNIX-based operating systems, MarkLogic runs as a Unix user named daemon.

In CentOS, to change which user MarkLogic will run as (e.g. to provide MarkLogic with the directory and file permissions of a particular user), the /etc/sysconfig/MarkLogic file can be edited, or a /etc/marklogic.conf file can be added.

The advantage of creating an /etc/marklogic.conf file, and putting any overrides in it, is that the /etc/marklogic.conf file survives MarkLogic upgrades.

In non-Unix operating systems (e.g. Windows), it is not necessary to specify a user for running MarkLogic.

Other Platforms

The Installation Guide for All Platforms (docs.marklogic.com/guide/installation) contains the steps for installing MarkLogic on each supported operating system.

Videos that demonstrate the installation steps for each platform are at developer.marklogic.com/video/data-hub (located half way down the web page).

Uninstalling MarkLogic

To uninstall MarkLogic, the first step is to stop it running. The next step is to do the uninstall steps for the platform that it is on. The steps for each platform are at docs.marklogic.com/guide/installation/procedures#id_53295.

If desired, once MarkLogic has been uninstalled, the remaining data and configuration files can be removed by deleting the /var/opt/MarkLogic directory (Linux), the C:\Program Files\MarkLogic folder (Windows), or the /Users/${user}/Library/Application Support/MarkLogic folder (Mac).
