Introducing Search

With search included in the original design of MarkLogic, MarkLogic has powerful real-time search capabilities, including the ability to query documents, triples, and/or rows. In addition, MarkLogic offers geospatial search, as well as alerting (i.e. reverse search).

MarkLogic's extensive search features include facets, type-ahead suggestions, highlighting, snippets, relevance ranking, document structure-based search, proximity boosting, search term weighting, wildcards, stemming, case sensitivity, punctuation sensitivity, diacritic sensitivity, and more.

For example, MarkLogic's documents, triples, and/or rows search capabilities could be used with geospatial search capabilities to create discovery applications. These applications could even include alerting, for notification of any new items.

Search APIs

For searching documents, cts.search, JSearch, search:search, the REST API /v1/search endpoint, the Java API, or the Node.js API are commonly used.

Each of the search APIs above is built upon the ones below it.


To search triples, SPARQL or the Optic API are commonly used.

To search rows, SQL or the Optic API are commonly used.
